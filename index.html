What's CSS?

  • If HTML is the bones of a web page defining its structural layout
  • and Javascript is the brains logically responding to user input
  • then CSS is the skin and clothes showing how it appears rendered by our browsers
  • CSS = Cascading Style Sheets

Problems with CSS

  • Graphic designers think it looks too much like programming, there are no built-in content creation tools
  • Programmers think it lacks any object-oriented structure promoting cut-and-paste spaghetti-code disasters
  • Scaling challenges in large projects
  • I can guess CSS was designed for laying out a single page document rather than an interactive application of any complexity
  • Doesn't encourage sharing best-practices/innovation across enterprise groups (modules/libraries etc.)

Scoping Out SASS

  • It's a meta language extending CSS wrapping it up into a blanket of comfy organization
  • Simply copy your .css files to .scss
  • Add whatever level of "SASS"ness you like to your newly created .scss files
  • The SASS converter turns the .scss files into .css files that your web page uses
  • SASS = Syntactically Awesome Style Sheets
  • Incorporate new SASS concepts into your project as the team becomes confident

SASS: The Best Parts

SASS has a ton of amazing features inherit to its language layout and held within its libraries

  • Mixins: encapsulate commonly used CSS patterns into little reusable snippets
  • Parameters: Pass variables into mixins and functions giving them infinite variety
  • Variables: Assign a named resource a color, or pixel size, then reference it; no hard-coding
  • Functions: concise bundles of logic for use over and over
  • Hierarchical Nesting: placing selectors "inside" selectors clearly shows relative ownership, inheritence, and specificity
  • Color Math: Modify colors by adding, subtracting, multiplication
  • File Fragments: break down big problems into smaller solutions, pulled in by a master using "@import" -> easier to manage and no extra http hits passed from server to client
  • Modern Comments: // I am a comment
  • Logical Conditions: @if, @for, @each, @while

I guarantee there's more awesomeness lurking in SASS. Check it out for yourself in their documentation.

Mixins

Creating mixins tucks away the ugly details of beautiful styles defining reusable blocks of selector rules. DRY. Encapsulation! Joy!

@mixin backgroundGradientLinear() {
  background: red
  background: -moz-linear-gradient(top, red 0%, blue 100%);
  background: -webkit-linear-gradient(top, red 0%, blue 100%);
  background: -o-linear-gradient(top, red 0%, blue 100%);
  background: -ms-linear-gradient(top, red 0%, blue 100%);
  background: linear-gradient(top, red 0%, blue 100%);
  filter: progid: DXImageTransform.Microsoft.gradient(
    startColorstr='red', endColorstr='blue', GradientType=0 );
}
.gradients-example {
  @include backgroundGradientLinear();
  border: 2px solid #000000;
  box-shadow: #000000 5px 5px 15px;
  -moz-box-shadow: #000000 5px 5px 15px;
  -webkit-box-shadow: #000000 5px 5px 15px;
  height: 100px;
  width: 400px;
}

I sense hard-coding.

Parameters

Parametric mixin definitions allow infinite variation.

@mixin backgroundGradientLinear($topColor, $bottomColor) {
  background: $topColor
  background: -moz-linear-gradient(top, $topColor 0%, $bottomColor 100%);
  background: -webkit-linear-gradient(top, $topColor 0%, $bottomColor 100%);
  background: -o-linear-gradient(top, $topColor 0%, $bottomColor 100%);
  background: -ms-linear-gradient(top, $topColor 0%, $bottomColor 100%);
  background: linear-gradient(top, $topColor 0%, $bottomColor 100%);
  filter: progid:DXImageTransform.Microsoft.gradient(
    startColorstr='$topColor', endColorstr='$bottomColor',
    GradientType=0);
}
@include backgroundGradientLinear(red, blue);

Crafted an excellent mixin? Document it! Share it! Have a coffee.

Variables

Hard-coding is generally bad form. Stuff changes. Choose a named resource, assign it a value, reference the resource.

Pick descriptive, self-documenting variable names. Favor shape/composition as naming inspiration rather than color.

$panel-background-color-1: #62707D;
$panel-background-color-2: #2F3A43;
$frame-color: #101010;
$shadow-color: #000020;

Refining

Leveraging what we know about variables and mixins, let's fine tune that previous "gradients" class selector.

.gradients-example {
  @include backgroundGradientLinear($panel-background-color-1,
    $panel-background-color-2);
  border: 2px solid $frame-color;
  @include boxShadow($shadow-color 5px 5px 15px);
  height: 100px;
  width: 400px;
 }

Now that's some Professional Engineering ®.

Functions

Functions look a lot like mixins, but return a single value used in the place it's called.

Example here takes two pixels values, divides them, and returns them as a percentage. I used it in the "fluid layout" examples you've seen.

@function fluidPercent($target-px, $context-px) {
  @return percentage($target-px / $context-px);
}

Calling this function might look like:

.hotel-info-popup {
  left: fluidPercent(75px, 960px);
  padding: 0;
  position: fixed;
  top: fluidPercent(65px, 960px);
  width: fluidPercent(795px, 960px);

Color Math

I concrete example using color math is making a shadow/tint of a base color for buttons.

$accept-clr-top: #7bbf81;
$accept-clr-bot: #0f9a1b;
$color-shift: 10%;
.simple-button {
  @include backgroundGradientLinear($accept-clr-top, $accept-clr-bot);
}

.simple-button:hover {
  @include backgroundGradientLinear(darken($accept-clr-top,
    $color-shift), darken($accept-clr-top, $color-shift));
}

.simple-button:active {
  @include backgroundGradientLinear(lighten($accept-clr-top,
    $color-shift), lighten($accept-clr-top, $color-shift));
}

Color Math Further

SASS types and libraries support more color math.

  • RGBA and HSL color spaces
  • hue, saturation, lightness,
  • adjust-hue, saturate, desaturate, grascale, invert, complement
  • opacify, transparentize, scale-color

@import

  • @import directive allows a style sheet to incorporate another one
  • CSS incurs additional http hits at run-time
  • SASS treats it as a pre-processor extension combining file fragments into a master container
  • Filenames starting with underscore are partials and SASS won't compile them into .css files
  • How to use them then? Choose a master file that includes those partials. For example "MyProject.scss" might begin:
    • @import "reset";
@import "buttons";
@import "forms";

Installation: Programmers

SASS is written in Ruby and bundled up for install. Assuming you have all that then this is easy:

"gem install sass"

Command-line compiler turns .scss into .css, and here's a helper macro to stick in your .profile.

alias sass_my_project = 'cd ~/Dev/MyProject/assets; sass --watch ./sass:./css'

Use it in an open terminal window. Moves to your resource directory, puts SASS compiler into "watch" mode. Assumes project directories something like this:

/MyProject/app/views/*.*
/MyProject/app/*.js
/MyProject/assets/css
/MyProject/assets/sass
/MyProject/assets/image
/MyProject/lib/*.js
